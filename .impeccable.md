## Design Context

### Users
Shinkai Docs is used by developers, technical evaluators, and product users who need fast access to setup guides, feature documentation, and API references. They are typically switching between product areas and expect the docs shell to stay stable while the left navigation changes by section.

### Brand Personality
Confident, polished, approachable. The interface should feel like a mature product documentation experience rather than a generic open-source template. The emotional goal is trust and clarity, with light product warmth from the coral Shinkai brand color.

### Aesthetic Direction
Preserve the existing Mintlify-based Shinkai Docs look and feel shown in the reference screenshots. The UI should stay light, editorial, and restrained: airy white backgrounds, thin dividers, compact top navigation, coral active states, soft rounded pills, and a strong documentation reading layout with left nav, content column, right TOC, and bottom Ask AI bar.

### Design Principles
1. Preserve shell parity with the current docs experience before introducing new styling ideas.
2. Keep section switching obvious: top tabs define the product area, left navigation changes within that area.
3. Favor clean reading ergonomics over decorative UI.
4. Use the coral brand accent sparingly for active states, emphasis, and primary actions.
5. Match interaction patterns users already learned from the current docs site.
